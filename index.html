Methodology and Tools

Task 1: Social Engineering Password Patterns

>Lab setup:

Environment: Local isolated lab running a Flask-based authentication server on 127.0.0.1:5000.

APIs tested:

  • /api/level1
  • /api/level2

Data source: Employee Profile.txt

Scope: Testing was limited strictly to the provided environment and accounts, with no interaction with real users or external systems.

Tools used: Python, NLKT, Flask test server, Requests.

>Methodology:

  • Term extraction: Relevant words and numbers were extracted from the employee profile using a scripted NLP-based approach. English stopwords were removed to reduce noise, producing a refined list of meaningful terms.
  • Credential generation: Username and password candidates were generated according to the strict rules defined for Level 1 and Level 2.
  • Brute-force execution: Python script iterated through all valid username/password combinations and submitted POST requests to the appropriate API endpoint until valid credentials were discovered.
  • Result analysis: Time taken, number of attempts, and credential complexity were recorded and analysed to understand how password design choices affected attack feasibility.

Task 2: Wiretap the Secret (packet capture) Forensics

>Lab setup:

Application: Fignal internal messaging platform.

Traffic capture: Live packet capture using Wireshark on the loopback interface.

Data source: Generated traffic from the provided Fignal server and client executables.

Scope: Analysis limited to the authorised lab environment provided for this assignment.

Tools used: Wireshark, Display filters, Follow TCP Stream, Screenshots, Excel, PowerShell.

>Methodology:

  • Captured network traffic while the Fignal client generated authentication and messaging activity.
  • Applied protocol and display filters to isolate HTTP traffic.
  • Inspected packet payloads and reassembled TCP streams to extract credentials and message content .
  • Documented security weaknesses based on observed behaviour.
  • Designed network and application-layer mitigations grounded in the findings.

Task 4: Membership Inference Attack

>Lab setup:

Environment: Python with PyTorch.

Dataset: CIFAR-10

OS: Windows

Tools used: PyTorch, Torchvision, NumPy, scikit-learn, tqdm.

The experiment followed a standard shadow model → attack model → evaluation pipeline.

>Methodology:

  • Shadow models were trained to mimic the target model’s behaviour.
  • Loss and prediction correctness were extracted as features.
  • An attack classifier was trained to distinguish between training (IN) and non-training (OUT) samples.
  • Initial results exposed a class imbalance flaw.
  • Balancing the dataset enabled accurate inference against the target model.
  • Final evaluation confirmed attack success on unseen data.

Conclusion

This project brought together a range of practical cybersecurity skills, combining technical investigation with structured analysis and clear documentation. By working through each task and room, a methodical approach was taken to understanding the target environments, defining simple threat models, and testing different attack paths. Failed attempts and pivots were an important part of this process, helping to refine techniques and reflecting how real-world security work is often iterative rather than linear.

Across the project, relevant tools and techniques were applied in realistic scenarios, leading to the successful recovery of the required flags and final secret string. These outcomes demonstrate not only technical effectiveness but also the importance of careful reasoning, patience, and evidence-based decision making.

A practical challenge arose during Task 4, where the workload proved to be resource-intensive and the primary laptop experienced significant performance limitations. To avoid delays and ensure the task could be completed accurately, an alternative computer was used. This allowed the work to continue efficiently without affecting the methodology or validity of the results.

Overall, the project strengthened hands-on cybersecurity skills, improved confidence with industry-relevant tools, and reinforced the importance of adaptability and clear reporting. The experience gained is directly applicable to future security testing and analysis work.
